# 【ATLAS機能評価】Phase 6 業務プロフィール

```
【ATLAS機能評価】

機能名：業務プロフィール・安全な情報保管・成果物差し込み基盤（Phase 6）
ユーザー価値：会社名・署名・連絡先などを構造化保存し、資料・メールへ繰り返し手入力せず差し込める
差別化：AI曖昧記憶ではなく権限付き構造化データ。不足時は needs_input で偽完成を防ぐ
繰り返し作業の削減：はい
AI必要度：低（CRUD・差し込み・検証は通常プログラム。抽出時のみAI）
AIなしで実装可能：大部分はい
運営コスト：中（テーブル・RLS・暗号化）
外部APIコスト：低（抽出時のみ）
コスト削減案：
  - エコモード：既存挙動を変更しない
  - まとめて生成：対象外
  - キャッシュ再利用：profile version をキャッシュキーに含め再生成を抑制
  - 予約実行：対象外
  - AI起動条件：抽出リクエスト時のみ
  - 外部API最小化：sanitizeContextForAI で最小項目のみ
  - 承認後実行：毎回確認・口座は確認後のみ
  - 再生成禁止：artifact_data_bindings で差し込み結果を記録
優先度：P0
```

既存の `lib/user-profile`（仕事の好み・localStorage）とは分離する。  
ユーザー向け名称は「業務プロフィール」。テーブルは既存 `projects` と衝突しないよう `atlas_business_*` を使う。
